# Integrations

## Adapter-first principle

External providers should be wrapped behind interfaces so the platform can swap providers without large rewrites.

## Current adapter categories

- maps and geocoding
- email
- SMS
- object storage
- feature flags
- payments
- portal syndication
- identity verification
- e-signature routing
- analytics

## Integration posture

- No scraping of competitor sites.
- Licensed datasets only for enrichment such as sold history, estimates, planning overlays, and comparable sales.
- Every external data surface should expose provenance in the UI.
