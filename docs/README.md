# Homeowner Documentation

## Overview

This documentation set turns the repository into an operational foundation for a private property sale company. It covers:

- product strategy and user flows
- jurisdiction-aware compliance rules
- system architecture and data model
- API contracts
- privacy, security, and retention policies
- deployment, backups, and observability
- legal drafts and help-centre content

## Documentation map

- `product/`: PRD, user flows, and state-based rules
- `architecture/`: system design, data model, integrations, media pipeline, and search/map notes
- `security/`: privacy-by-design, retention, and incident response
- `ops/`: deployment, observability, and recovery
- `api/openapi.yaml`: current public API draft
- `testing/test-plan.md`: required quality strategy
- `ux/content-guide.md`: content and trust-writing principles
- `legal/`: draft legal and policy documents
- `help/`: customer-facing education and guidance
